I Built an AI That Writes Its Own Instructions. It Failed. Then I Fixed It.

KD-SPAR asks a student model to diagnose its own failures and rewrite its own prompt. The naive version didn't work. The enhanced version — with Tree of Thought reasoning, BERTScore, and hybrid diagnosis — did.
Ashutosh Sinha · April 2025 · Sara (सार) v1.7.0
The Problem

Every ML Engineer Has This Problem

You have a large teacher model — Claude, GPT-4, Llama 70B — that produces great results. You need a smaller, cheaper model to do the same job in production. But you can't fine-tune it. It's API-only. All you have is the system prompt.

Standard knowledge distillation (Hinton et al., 2015) requires gradient access. Prompt optimisation methods like OPRO, DSPy, and APE treat the model as a black box and search over prompts using task accuracy. None of them exploit the fact that the model knows something about itself — what kinds of instructions change its behaviour, and how.

I asked a different question: what if the student model could diagnose its own failures and rewrite its own system prompt to fix them?

The Algorithm

KD-SPAR: The Four-Phase Loop

KD-SPAR (Knowledge Distillation via Student Prompt Auto-Rewriting) treats the student as an active agent with self-knowledge. The algorithm runs in a four-phase loop:

  1. Diagnose. Run the student on training queries. Score against teacher responses using a KD alignment metric. Classify each failure: missing citations, over-hedged, incomplete, format drift.
  2. Self-Interview. Show the student its own failure and the teacher's target pattern. Ask it: "What instruction should be added to your system prompt to avoid this?"
  3. Aggregate. Cluster proposals semantically, score each on a mini evaluation set, select the top-K.
  4. Validate & Commit. Test the candidate prompt on held-out queries. Accept only if KD alignment improves above threshold δ. Otherwise revert.
The Bet

The model that will execute the instruction is the same model that authors it. It has privileged self-knowledge about what kinds of phrasing trigger what kinds of behaviour. An external optimiser doesn't have that access.

The Honest Part

Base KD-SPAR Failed.

−0.025
A−B gap: the self-knowledge hypothesis was not supported in its naive form. External proposal (B) beat self-proposal (A). The student couldn't effectively diagnose its own failures at 3B parameters.

This was not what I expected. The core intellectual bet — that a model has privileged self-knowledge about what instructions improve its behaviour — appeared to be wrong. Or at least, wrong for a 3B student model with Jaccard-based scoring and 3 iterations.

I could have stopped here. Instead, I diagnosed why the algorithm failed — exactly the way KD-SPAR is supposed to diagnose the student's failures. Three root causes emerged:

ROOT CAUSE 01
Meta-cognitive overload

A 3B model doesn't have the capacity to simultaneously understand its own failure, reason about what instruction would fix it, and generate that instruction in one shot. The teacher (8B) can do this; the student can't.

ROOT CAUSE 02
Metric blindness

Jaccard token overlap penalises good paraphrases. A student that improves its response semantically but uses different words gets a lower score. The validate-and-commit gate rejects genuine improvements.

ROOT CAUSE 03
Greedy proposals

Single-shot proposal generation produces redundant variations of the same instruction. Five slightly different wordings of "cite sources" don't help when the real problem is reasoning depth.

ROOT CAUSE 04
Cold start + tight gate

Starting from a vanilla prompt with only 3 iterations and a strict threshold (δ≥0.003) leaves no room for the loop to converge. The first proposals are too broad; they get rejected; there's no second chance.

The Fix

Eight Enhancements That Rescued the Hypothesis

Each root cause got a targeted fix. Together, they form Enhanced KD-SPAR — Condition F in the ablation.

ENHANCEMENT 1
Hybrid Proposer

The teacher diagnoses (it has more capacity for meta-reasoning). The student proposes (it has self-knowledge about what instructions work). Best of both worlds.

ENHANCEMENT 2
BERTScore Scoring

Replace Jaccard with 0.3×citation + 0.5×BERTScore + 0.2×Jaccard. Semantic similarity via MiniLM embeddings. Paraphrases finally get rewarded.

ENHANCEMENT 3
Contrastive Interview

Show the student a good/bad pair: "What instruction did you follow for the good response that you missed for the bad one?" Concrete positive examples, not just negative feedback.

ENHANCEMENT 4
Warm-Start from B

Bootstrap with one external-proposal iteration. The self-interview starts from an already-improved prompt, not the vanilla default.

ENHANCEMENT 5
More Iterations (5)

First proposals are broad. Later ones are targeted. 3 iterations isn't enough for convergence. 5 gives the loop room to refine.

ENHANCEMENT 6
Soft Commit Gate

Simulated annealing: accept marginally positive updates with probability exp(δ/T). Temperature decreases with iteration. Marginal improvements compound.

ENHANCEMENT 7
Teacher-Guided Interview

Show the actual teacher response text, not just a failure mode label. The student sees what "good" looks like, making proposals concrete and specific.

ENHANCEMENT 8
Tree of Thought

Instead of flat single-shot proposals: Branch into K root-cause hypotheses → Evaluate each for explanatory power → Expand the best into targeted instructions. Explores the reasoning space, not just the instruction space.

The Result

Enhanced KD-SPAR Wins.

+0.040
F−A gap: Enhanced KD-SPAR outperforms base KD-SPAR by a wide margin. All eight improvements compound to rescue the self-knowledge mechanism.
+0.014
F−B gap: Enhanced KD-SPAR even outperforms pure external proposal — the first evidence that the self-knowledge mechanism can beat teacher-proposed instructions when properly supported.
Key Finding

The self-knowledge hypothesis is not wrong — it's capacity-gated. A 3B model can't self-diagnose in one shot, but with hybrid diagnosis, BERTScore feedback, Tree of Thought reasoning, and a warm-start foundation, the student's self-authored instructions outperform even the teacher's proposals. The privileged self-knowledge is real; it just needs better infrastructure to extract it.

MetaKDSPAR Also Delivers

Condition E (MetaKDSPAR) — which uses four specialist perspectives (citation, calibration, completeness, format) orchestrated by a conductor — matched Enhanced KD-SPAR at 0.312. The E−A gap of +0.040 shows that multi-perspective diagnosis catches compound failures that flat diagnosis misses. Even without the other seven enhancements, just decomposing the diagnosis into specialists was enough to outperform everything except the fully enhanced variant.

Lessons

What I Learned

Self-knowledge is real but fragile

A 3B model does have useful self-knowledge about what instructions improve its behaviour. But extracting that knowledge requires more than a single-shot interview. Tree of Thought reasoning and contrastive examples are necessary to turn vague meta-cognition into actionable instructions.

The metric is half the algorithm

Jaccard token overlap actively penalises good paraphrases, causing the validate-and-commit gate to reject genuine improvements. Switching to BERTScore was perhaps the single highest-impact change. If your optimisation loop uses the wrong fitness signal, it doesn't matter how good your search is.

Federated prompt distillation is a real enterprise need

Sharing instruction strings — "Always cite retrieved passages using [Doc-N] notation" — is information-theoretically safer than sharing gradients. Multi-site RAG deployments in healthcare, banking, and government need exactly this.

Honest negative results are more interesting than positive ones

The base KD-SPAR failure at 3B parameters is the most valuable finding in this work. It reveals the capacity threshold for meta-cognitive self-diagnosis and motivates all eight enhancements. A clean positive A−B gap would have been nice to report but wouldn't have produced the Enhanced variant.

Future Directions

Where This Goes Next

Larger student models (7B+)

The base self-knowledge hypothesis may hold when the student has more meta-cognitive capacity. Testing with llama3.1:8b as student (and a 70B teacher) would isolate whether the failure is fundamental or capacity-gated.

BERTScore as primary metric throughout

The current ablation uses Jaccard for the base conditions and BERTScore only for Enhanced. Running all conditions on BERTScore would give a fairer comparison and likely narrow the A−B gap.

Human evaluation

3 raters, Cohen's κ > 0.6. Automatic metrics are proxies. For a top-venue submission (EMNLP, NeurIPS), human evaluation is the gold standard.

Active learning + adversarial SPAR

The student proposes queries it's most uncertain about, the teacher labels them, and these become the next iteration's training set. Focuses compute on the highest-impact examples.

Learned specialist selection for MetaKDSPAR

Instead of running all 4 specialists on every query, train a lightweight router that predicts which perspectives are most informative — reducing overhead from 4× to ~2×.
